Guía de Control de Versiones y GIT

¿Qué es un control de versiones?

Un sistema de control de versiones (VCS) es una herramienta que ayuda a gestionar los cambios en los archivos de un proyecto a lo largo del tiempo. Permite realizar un seguimiento de las modificaciones, revertir a versiones anteriores y colaborar con otros usuarios.

¿Qué es GIT?

GIT es un sistema de control de versiones distribuido que permite a los desarrolladores gestionar el historial de cambios en el código fuente. Fue creado por Linus Torvalds en 2005 y es conocido por su rapidez y eficiencia en el manejo de proyectos grandes.

¿Qué es un repositorio?

Un repositorio es un espacio donde se almacena y gestiona el código fuente de un proyecto. Hay dos tipos principales:

  • Repositorio local: Es una copia del proyecto almacenada en el sistema de archivos de un usuario.
  • Repositorio remoto: Es una versión del proyecto almacenada en un servidor remoto, accesible a través de Internet.

¿Qué es GITHUB?

GitHub es una plataforma de alojamiento de código basada en la web que utiliza Git para el control de versiones. Ofrece herramientas para la colaboración, la gestión de proyectos y el alojamiento de repositorios.

¿Qué es una rama en GIT y para qué sirve?

Una rama en GIT es una línea de desarrollo independiente que permite trabajar en diferentes características o correcciones de un proyecto sin afectar la rama principal (generalmente llamada "main" o "master"). Las ramas facilitan la colaboración y el trabajo en paralelo.

¿Cuáles son los estados en GIT?

Los estados en GIT son:

  • Modificado: El archivo ha sido editado, pero no ha sido añadido al área de preparación.
  • Preparado: El archivo está en el área de preparación y listo para ser incluido en el próximo commit.
  • Confirmado: El archivo ha sido guardado en el historial de commits.

¿Qué es la terminal, consola o línea de comandos y para qué sirve?

La terminal o línea de comandos es una interfaz que permite a los usuarios interactuar con el sistema operativo mediante la entrada de comandos. Es útil para realizar tareas administrativas, ejecutar programas y gestionar archivos de manera eficiente.

Funcionamiento de comandos

  • ls: Muestra una lista de archivos y directorios en el directorio actual.
  • cd: Cambia el directorio actual.
  • cd ..: Sube un nivel en la jerarquía de directorios.
  • pwd: Muestra la ruta del directorio actual.
  • mkdir: Crea un nuevo directorio.
  • touch: Crea un nuevo archivo vacío o actualiza la fecha de modificación de un archivo existente.
  • rm: Elimina archivos o directorios.
  • cp: Copia archivos o directorios. Ejemplo: cp archivo1.txt archivo2.txt copia archivo1.txt a archivo2.txt.
  • mv: Mueve o renombra archivos o directorios. Ejemplo: mv archivo1.txt nuevo_directorio/ mueve archivo1.txt al directorio nuevo_directorio.

Listado de comandos de GIT usados en clase

  • git init: Inicializa un nuevo repositorio Git.
  • git add: Añade archivos al área de preparación.
  • git commit: Guarda los cambios en el historial de commits.
  • git push: Envía los commits al repositorio remoto.
  • git pull: Recupera y fusiona los cambios del repositorio remoto.
  • git branch: Muestra, crea o elimina ramas.
  • git status: Muestra el estado de los archivos en el repositorio.
  • git clone: Clona un repositorio remoto a tu máquina local.
  • git remote add origin [URL]: Conecta tu repositorio local a un repositorio remoto.
  • git config --global user.name "[nombre]": Configura tu nombre de usuario global.
  • git config --global user.email "[email]": Configura tu correo electrónico global.
  • git config --global credential.helper cache: Guarda tus credenciales temporalmente.
  • git config --global credential.helper store: Guarda tus credenciales de forma permanente.
  • git ls-remote [URL]: Verifica la conexión y las credenciales con el repositorio remoto.
